ðŸ“˜ Operator Precedence and Associativity

When an expression has more than one operator, the program needs rules to decide which operator to perform first. Operator precedence is the rule that defines this order. Think of it like the order of operations in math (PEMDAS/BODMAS): multiplication and division come before addition and subtraction. In C#, operators have a predefined precedence.

For example, the multiplication operator * has higher precedence than addition +. Consider the C# statement:

int x = 4 + 3 * 5;

Even though + appears first, the multiplication is done first because it has higher precedence. This means 3 * 5 is calculated to give 15, and then we add 4, resulting in x = 19. If addition had been done first (ignoring the rules), we would have added 4+3=7 and then multiplied by 5 to get 35, which is not what C# does.

In short, multiplication and division happen before addition and subtraction. You can change the order explicitly by using parentheses, just like in math. For example:

int x = (4 + 3) * 5; // result = 35

Parentheses always force that part to be done first, regardless of the normal precedence.

ðŸ”„ Operator Associativity

Sometimes the order depends on associativity. Most operators (like +, -, *, /) are left-to-right. For example:

int result = a * b / c; // evaluated as (a * b) / c

But assignment operators work right-to-left. For example:

int a, b, c; a = b = c = 3; // c=3, then b=c, then a=b

Result: all three variables become 3.
